If you spend a significant amount of time editing text on a daily basis, it's probably worth your time to learn how to use a good text editor, and as far as text editors go, I don't think they make them much better than Vim. Vim is an extremely powerful, flexible, open-source text editor. However, along with all that power and flexibility comes a moderate learning curve. In my experience, it's hard to come across a solid beginner's guide to Vim. Many guides only lay out the most basic commands, without inspiring the reader to use Vim over another text editor, such as Gedit or Notepad. Other guides will list every possible command, with no comment on which commands are more useful than others, or when they are most useful. In this guide, I will discuss how I use Vim. I will explain which commands I use most, and how I use them.

I don't actually use Vim, per se, unless I'm quickly editing a file at the command line. For most purposes, I use Sublime Text with Vim keybindings, but many other editors will allow you to use Vim keybindings as well. Honestly, I think that the Vim keybindings are more important than the specific editor that implements them. Use whichever editor makes you happiest.

Changing Modes

Most guides to Vim start out by discussing movement, but that seems silly to me. There's no sense in trying to move around a document if you haven't typed anything yet. So first, let's talk about modes. The two main modes in Vim are command mode (sometimes called normal mode), and insert mode, where you can actually type letters and insert them into your document. There are three main commands to get into insert mode from command mode:

  1. a (append)
  2. i (insert)
  3. o (open).
a will enter insert mode after the cusor. i will enter insert mode before the cursor. o will create a new, blank line between the current line and the one below it, and enter insert mode there. I never write anything from top to bottom, but rather I jump around, inserting text here and there, so I use o quite a bit. All three of these commands can be modified by capitalizing them, but a and i are the most exciting when modified. A and I, instead of entering insert mode after or before the cursor, will enter insert mode at the end or start of the current line. I use these two commands all the time. O will create a blank line and enter insert mode between the current line and the line above it, rather than the one below the current line. This command is nice to know, but I don't use it all that often.

To go back to command mode, there is only one command. Simply press escape.

Every guide to Vim will tell you that escape is the default command to return to command mode, but no one actually keeps that default setting. Before you even finish reading this guide, I strongly recommend that you remap that command to something more comfortable, since you will probably end up typing it every two to ten seconds. Many QWERTY typists use the key sequence j j to enter command mode, so they don't even have to leave the home row. It's also good because there are no English words with two consecutive j's in them, so you are unlikely to accidentally enter command mode while inserting text. Personally, I like to roll my fingers across f d s. Just pick something you can type quickly and comfortably.
