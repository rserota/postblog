If you spend a significant amount of time editing text on a daily basis, it's probably worth your time to learn how to use a good text editor, and as far as text editors go, I don't think they make them much better than Vim. Vim is an extremely powerful, flexible, open-source text editor. However, along with all that power and flexibility comes a moderate learning curve. In my experience, it's hard to come across a solid beginner's guide to Vim. Many guides only lay out the most basic commands, without inspiring the reader to use Vim over another text editor, such as Gedit or Notepad. Other guides will list every possible command, with no comment on which commands are more useful than others, or when they are most useful. In this guide, I will discuss how I use Vim. I will explain which commands I use most, and how I use them.

I don't actually use Vim, per se, unless I'm quickly editing a file at the command line. For most purposes, I use Sublime Text with Vim keybindings, but many other editors will allow you to use Vim keybindings as well. Honestly, I think that the Vim keybindings are more important than the specific editor that implements them. Use whichever editor makes you happiest.

Changing Modes

Most guides to Vim start out by discussing movement, but that seems silly to me. There's no sense in trying to move around a document if you haven't typed anything yet. So first, let's talk about modes. The two main modes in Vim are command mode (sometimes called normal mode), where you can type commands to move around and edit your document, and insert mode, where you can actually type letters and insert them into your document. There are three main commands to get into insert mode from command mode:

  • a (append)
  • i (insert)
  • o (open)
a will enter insert mode after the cusor. i will enter insert mode before the cursor. o will create a new, blank line between the current line and the one below it, and enter insert mode there. I never write anything from top to bottom, but rather I jump around, inserting text here and there, so I use o quite a bit. All three of these commands can be modified by capitalizing them, but a and i are the most exciting when modified. A and I, instead of entering insert mode after or before the cursor, will enter insert mode at the end or start of the current line. I use these two commands all the time. O will create a blank line and enter insert mode between the current line and the line above it, rather than the one below the current line. This command is nice to know, but I don't use it all that often.

To go back to command mode, there is only one command. Simply press escape.

Every guide to Vim will tell you that escape is the default command to return to command mode, but no one actually keeps that default setting. Before you even finish reading this guide, I strongly recommend that you remap that command to something more comfortable, since you will probably end up typing it every two to ten seconds. Many QWERTY typists use the key sequence j j to enter command mode, so they don't even have to leave the home row. It's also good because there are no English words with two consecutive j's in them, so you are unlikely to accidentally enter command mode while inserting text. Personally, I like to roll my fingers across fds. Just pick something you can type quickly and comfortably.

Movement

Now that you can actually type text on your screen, let's learn how to move around. The basic movement commands are:

  • h(left)
  • j(down)
  • k(up)
  • l(right)
Honestly, I don't use these commands all that often. There are generally more precise ways to move to wherever you want to be. The movement commands I use most are:
  • w(word)
  • e(end)
  • b(back)
w will move you forward to the beginning of the next word. e will move you forward to the next end of a word.b will move you backward to the next beginning of a word. Note that Vim's definition of 'word' is a little strange. A string of letters and/or numbers is a word, and a string of punctuation is also a word. For example, the character string "a;bc.*" is actually six words. These three commands can all be modified by capitalizing them. When a word-based movement command is modified, it changes Vim's definition of a word to a more strict definition. When modified, w, e, and b will consider whitespace the only thing that seperates words. For example, the character string "a;b c.*" would be two words.

You can also move by paragraphs in Vim. Press { or } to move backwards or forwards by paragraph. I use these commands quite frequently when I'm debugging or simply reading code, since it allows me to jump around by conceptual chunks.

All of the movement commands I've mentioned so far can also be modified by putting a number before them, to repeat the movement that many times. For example, 7w will move the cursor seven words forward. Honestly, I don't do this very much. If I needed to move forward seven words, I would rather tap w until I got where I wanted to be, rather than count how many words I need to move by.

If you have line numbers visible in your text editor, you can take advantage of them and jump straight to specific lines using the G or gg commands (they're equivalent in this case). For example, to jump straight to line 23, type 23G or 23gg. I find that this is generally the simplest way to the line I'm looking at. If these two commands are not preceded by a number, gg will take you to the first line of the file, and G will take you to the last line of the file.

The last two motion commands that I'll mention here are $ and 0 (zero). $ moves you to the end of the line, and 0 moves oyu to the beginning of the line. I use these commands occasionally. However, usually when I want to go to the end or beginning of a line, it's because I want to insert text there, in which case A or I would be more precise.

Many programming languages give the programmer some flexibility in how they use whitespace, and many of these commands rely on whitespace to determine where to move. If you are consistent and mindful in your usage of whitespace when writing code, it will make the document easier to navigate.

Delete, Cut, Copy, Paste, Undo

To simply delete one character at a time, use x. This will delete the character under the cursor, making it function like a forward-delete (not backspace). If this command is modified to X, it will first move the cursor back one space and then delete, making it function more like a traditional backspace.

To cut text in Vim, use d (this command is technically called "delete"). This command must be combined with a motion that specifies what to delete. I often use this to cut a few words at a time, such as with d2w. It's also great to combine with paragraph motions (d{ or d}). Other handy delete commands are d0 and d$, which cut from the cursor to the end or beginning of the line, respectively. Similarly, dd and D will cut the entire current line. My favorite way to use d is with G, which will cut all the text from the location of the cursor until the specified line. For example, d22G would cut all the text from the cursor till line 22. I find this to be a very easy way to precisely select large amounts of text.

To copy text in Vim, use y. Using Vim-speak, this is referred to as "yanking" text. No one has ever been able to explain to me why it's called yanking, beyond the fact that it is done with the "y" key. y works the same way as d, and can be modified in all the same ways, except that it copies text instead of cutting it.

If you're not exactly sure how to precisely select an awkwardly sized chunk of text, you may find it useful to use visual mode. Press v to enter visual mode. In visual mode, you can use various motion commands to move around, and all text will be highlighted between the current location of the cursor, and where the cursor was when you entered visual mode. This is similar to holding shift and moving around with the arrow keys in an ordinary text editor. Once you've selected the text you wanted, you can cut or copy the whole selection with d or y. To exit visual mode, press v again.

Once you've cut or copied some text to the clipboard, you can paste it with p. By default, this inserts the text from the clipboard after the cursor. If modified to P, this command will insert the text from the clipboard before the cursor. Technically, this command is called "put", though I always think of it as "paste".

To undo an action, just press u. To redo an undone action, press ctrl+r (even if you're on a mac, redo uses control, not command).

There isn't actually any true 'delete' command in Vim. Anything that removes text actually puts it in the clipboard. Just be mindful that if you cut a large chunk of text with d, delete one character with x, and then attempt to paste with p, Vim will paste in the one character you deleted with x. The large selection you cut with d will be lost. Many implementations of Vim support multiple clipboards, so you can cut/copy multiple selections of text, and then paste them into your document in whatever order you like. I don't use this feature, but some people find it very useful.

Searching and Scrolling

There are two final things you should know about to round out your knowledge of Vim: searching and scrolling. To start a search, press /, type in some text to search for, then hit enter. Press n to move forwards through the search results, and press N to move backwards through the results. This is how I use search in Vim, but it is actually much more powerful than what I've described. If you would like to be a sophisticated searcher, you should look up more information on searching in Vim to find out tricky ways to craft your queries.

Scrolling commands are not really essential to know, but I really like them. zt will scroll your document so that the cursor is at the top of the screen. zb will scroll your document so that the cursor is at the bottom of the screen. zz will scroll your document so that the cursor is in the middle of the screen.

In Sublime Text, there is an option to enable scrolling beyond the bottom of the document. I like using that option so that I can type in the middle of the screen, even when I'm at the bottom of the document.

Now You Know Vim!

Those are all of the commands that I use in Vim. I feel like a pretty competent Vim user, and if you followed this guide, you should now be a pretty competent Vim user too. I know that this guide is a little thick for a beginner's guide, and yet falls short of a comprehensive manual. My hope is that after reading this guide, using Vim will be faster for you than not using Vim.
